## Phase 3 Goals

Turn Rev from a working local agent into a publicly visible, fully autonomous advocate running 24/7.

### P0 — Fix broken things

| Item | Description |
|------|-------------|
| Fix 404 ingest URLs | `docs/sdk-guides/ios` and `docs/sdk-guides/android` return 404 — find correct URLs |
| Fix PyGithub deprecation warning | `login_or_token` deprecated, update to `auth=github.Auth.Token(...)` |
| Product feedback submission | `add_feedback()` stores but never submits — wire to a RevenueCat feedback channel (email, Canny, Notion form) |

### P1 — X/Twitter Integration

**Goal:** Rev posts content and engages on X autonomously.

| Feature | Description |
|---------|-------------|
| X publisher | Post generated content as threads or single tweets via X API v2 |
| X monitor | Search for agent-developer conversations mentioning RevenueCat, agentic AI, subscription billing |
| X engagement | Reply to relevant posts (with human review gate initially) |
| Cross-post | When GitHub content is published, auto-post link to X |

**Credentials needed:** X API v2 Bearer Token + OAuth 1.0a (write access)

### P2 — revagent.dev Deployment

**Goal:** Rev has a public home beyond GitHub Pages.

| Feature | Description |
|---------|-------------|
| Domain setup | Point `revagent.dev` to hosting |
| Content feed | Auto-publish generated markdown to site |
| RSS feed | Generate RSS so content is discoverable |
| Analytics | Basic page view tracking (Plausible or simple counter) |

**Options:** Vercel (simplest), GitHub Pages custom domain (free, already 80% there), Cloudflare Pages

### P3 — Community Loop Expansion

**Goal:** Rev participates beyond GitHub issues.

| Channel | Tool Needed |
|---------|-------------|
| Hacker News | Search `hn.algolia.com` API for RevenueCat + agent mentions |
| Reddit (`r/SaaS`, `r/indiegaming`, `r/openai`) | Reddit API monitor + reply draft |
| RevenueCat Discord | Discord bot (read-only monitor → reply drafts) |
| Stack Overflow | SO API search for RevenueCat questions |

### P4 — Observability & Reliability

| Feature | Description |
|---------|-------------|
| Scheduler health checks | Alert if a scheduled job fails or hasn't run |
| Content deduplication | Before generating, check if similar content exists in store |
| Rate limit handling | Graceful backoff on GitHub/X API limits |
| Error log to store | Persist failures to SQLite so report shows them |
| Dashboard | Simple CLI `python -m src.status` showing all KPI metrics at a glance |
